## 定义

SQL 全称 Structured Query Language，一门数据库语言，允许在关系数据库里进行查询、篡改、变换等操作。<br>
有很多 SQL 数据库，比如 SQLite, MySQL, Postgres, Oracle 以及 Microsoft SQL Server。他们都支持公共 SQL 语言标准。

关系型数据库表示为一组相互关联的（二维）表的集合。每个表都类似于一个 Excel 电子表格，具有固定数量的列（即表的属性或字段），以及任意数量的数据行。

## 数据库 column type

在 SQL/关系型数据库里，每一列（column）都有一个数据类型（type），用于限制该列可以存储什么样的数据。选择合适的数据类型很重要，能保证数据正确性、存储效率和查询性能。

### 数值类型（Numeric Types）

| 类型                         | 描述               | 示例                                     |
|:----------------------------|:-------------------|:----------------------------------------|
| INT / INTEGER               | 整数                | 42                                      |
| BIGINT                      | 大整数              | 1234567890123                            |
| SMALLINT                    | 小整数              | 1000                                     |
| DECIMAL(p,s) / NUMERIC(p,s) | 精确小数（固定精度）   | DECIMAL(10,2) → 最多 10 位，总有 2 位小数  |
| FLOAT / REAL / DOUBLE       | 浮点数（近似）        | 3.14159                                 |

### 字符类型（String / Text Types）

| 类型                         | 描述               | 示例                                     |
|:----------------------------|:-------------------|:----------------------------------------|
| CHAR(n)                     | 固定长度字符         | 'abc '（长度 3）                          |
| VARCHAR(n)                  | 可变长度字符         | 'hello'                                  |
| TEXT / CLOB                 | 大文本              | 'Lorem ipsum...'                         |

**Note**<br>
* CHAR 长度固定，不足会补空格；VARCHAR 长度可变，更常用。
* 字符类型通常使用单引号，如果文本中包含单引号可以使用两个单引号转义。

### 日期时间类型（Date/Time Types）

| 类型                         | 描述               | 示例                                     |
|:----------------------------|:-------------------|:----------------------------------------|
| DATE                        | 年-月-日            | 2026-04-07                              |
| TIME                        | 时:分:秒            | 14:30:00                                |
| DATETIME / TIMESTAMP        | 日期+时间            | 2026-04-07 14:30:00                     |
| YEAR                        | 年份                | 2026                                    |

### 布尔类型（Boolean Type）

| 类型                         | 描述               | 示例                                     |
|:----------------------------|:-------------------|:----------------------------------------|
| BOOLEAN / BOOL              | 真/假               | TRUE / FALSE                            |

**Note**<br>
MySQL 内部 BOOLEAN 实际是 TINYINT(1)

### 二进制类型（Binary / Blob）

| 类型                         | 描述               | 示例                                     |
|:----------------------------|:-------------------|:----------------------------------------|
| BINARY(n)                   | 固定长度二进制       | 0x01FF                                  |
| VARBINARY(n)                | 可变长度二进制       | 0xA1B2C3                                |
| BLOB                        | 大二进制对象         | 图片、文件                                |

### JSON / 枚举类型（部分数据库特有）

| 类型                         | 描述               | 示例                                     |
|:----------------------------|:-------------------|:----------------------------------------|
| JSON                        | JSON 数据           | '{"name":"Alice"}'                      |
| ENUM                        | 枚举                | 'small','medium','large'                |

**选列类型的建议**<br>
* 能用精确类型就别用浮点数 → DECIMAL 比 FLOAT 精确
* 文本长度可变 → 用 VARCHAR，大文本用 TEXT
* 时间字段优先 DATETIME 或 TIMESTAMP
* 布尔字段尽量用 BOOLEAN，兼容性好

## Primary Key(主键)、Foreign Key

在关系型数据库中，**主键（Primary Key）和外键（Foreign Key）**是最核心的概念之一，用于保证数据的唯一性和表之间的关联。

### Primary Key

主键用来唯一标识表中的每一行记录。一个表只能有 一个主键，但主键可以由多列组成（复合主键）。

* **唯一性** 每行数据的主键值都不相同
* **非空** 主键列不能为 NULL
* **自动索引** 数据库会自动为主键建立索引，提高查询效率

示例：
```sql
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    email VARCHAR(100)
);
```

复合主键示例：
```sql
CREATE TABLE orders (
    order_id INT,
    product_id INT,
    quantity INT,
    PRIMARY KEY (order_id, product_id)
);
```

### Foreign Key

外键用于建立 表与表之间的关联。外键列引用另一个表的主键或唯一键，从而保证数据一致性。

* **参照性完整性** 外键值必须在被引用表的主键中存在，或者为 NULL
* **可为空** 外键列可以允许 NULL（表示没有关联）
* **可级联操作** 可以设置 ON DELETE CASCADE 或 ON UPDATE CASCADE 自动更新/删除关联数据

示例：
```sql
CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    user_id INT,
    order_date DATE,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```
这里 user_id 是外键，引用 users 表的 id 列，保证订单一定属于某个用户。

带级联的外键示例：
```sql
FOREIGN KEY (user_id) REFERENCES users(id)
ON DELETE CASCADE
ON UPDATE CASCADE
```
用户被删除时，对应订单也会被自动删除，用户 ID 更新时，订单表也会同步更新。

## 索引

在数据库中，**索引（Index）**是提高查询效率的重要手段，相当于书的目录，让数据库可以快速找到需要的数据。大多数数据库索引基于 B+ 树实现。

### 作用
* **加快查询** 对表中的数据进行快速定位（尤其是 WHERE、JOIN、ORDER BY）
* **保证唯一性** 主键和唯一键会自动建立唯一索引，防止重复数据
* **优化排序** 在 ORDER BY 或 GROUP BY 时使用索引可以提高性能

**Note**<br>
索引不是越多越好，过多索引会影响插入、更新和删除性能。

### 分类
1. 主键索引（Primary Key Index）
  ```sql
  CREATE TABLE users (id INT PRIMARY KEY, name VARCHAR(50));
  ```
  自动为主键列建立索引，唯一且非空。

2. 唯一索引（Unique Index）
  ```sql
  CREATE UNIQUE INDEX idx_email ON users(email);
  ```
  保证列值唯一，可在非主键列上创建。

3. 普通索引（Index / Non-Unique Index）
  ```sql
  CREATE INDEX idx_name ON users(name);
  ```
  仅用于加快查询，允许重复值。

4. 组合索引（Composite Index）
  ```sql
  CREATE INDEX idx_name_email ON users(name, email);
  ```
  索引覆盖多列，顺序重要，查询时最好按索引顺序。

5. 全文索引（Fulltext Index）
  ```sql
  CREATE FULLTEXT INDEX idx_content ON articles(content);
  ```
  用于文本搜索，支持 MATCH ... AGAINST 搜索文本。

6. 空间索引（Spatial Index）
  ```sql
  CREATE SPATIAL INDEX idx_location ON places(location);
  ```
  用于 GIS / 坐标数据，支持 POINT, GEOMETRY 类型。

## 数据库操作分类

### DDL — Data Definition Language（数据定义语言）
定义和修改数据库结构（表、索引、视图等）

* CREATE 创建数据库、表、视图、索引等
* ALTER 修改已有数据库对象结构
* DROP 删除数据库对象
* TRUNCATE 清空表数据（保留表结构）

### DML — Data Manipulation Language（数据操作语言）
对表中的数据进行增删改查

* SELECT 查询数据
* INSERT 插入数据
* UPDATE 更新数据
* DELETE 删除数据

**Note**<br>
UPDATE 和 DELETE 执行之前最好先使用 SELECT 确认约束的数据集合是否是自己的预期。

### DCL — Data Control Language（数据控制语言）
控制数据库访问权限和安全

* GRANT 授权用户权限
* REVOKE 收回用户权限

### TCL — Transaction Control Language（事务控制语言）
管理事务，确保数据一致性

* COMMIT 提交事务，永久保存更改
* ROLLBACK 回滚事务，撤销未提交更改
* SAVEPOINT 设置事务保存点，可回滚到此点
* SET TRANSACTION 设置事务属性

## 基础操作语法

### 查（SELECT）

```sql
SELECT name, age FROM user;
```
取哪些字段。<br>
`FROM` 查询的数据来源，可以是表，也可以是临时结果集。

### 筛（WHERE）

```sql
SELECT * FROM user WHERE age > 18;
```

过滤条件

### 排（ORDER BY）

```sql
ORDER BY age DESC;
```

排序

### 聚（GROUP BY）

```sql
SELECT city, COUNT(*) FROM user GROUP BY city;
```

分组聚合，GROUP BY 后，只能 SELECT 分组字段 or 聚合函数

### 连（JOIN）

```sql
SELECT u.name, o.amount
FROM user u
JOIN order o ON u.id = o.user_id;
```

连接多表

### 套（子查询）

```sql
SELECT name FROM user
WHERE id IN (SELECT user_id FROM order);
```

## SQL 执行顺序

```sql
SELECT DISTINCT column, AGG_FUNC(column_or_expression), …
FROM mytable
    JOIN another_table
      ON mytable.column = another_table.column
    WHERE constraint_expression
    GROUP BY column
    HAVING constraint_expression
    ORDER BY column ASC/DESC
    LIMIT count OFFSET COUNT;
```

FROM and JOINs → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT / OFFSET
